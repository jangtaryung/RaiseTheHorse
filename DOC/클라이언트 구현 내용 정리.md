### 전투 참조 통합: ChariotStats → Chariot 직접 참조로 전환

모든 전투 코드가 ChariotStats를 거치지 않고 Chariot 모델을 직접 참조하도록 통일.

| 항목 | 기존 | 변경 후 |
|------|------|---------|
| 전투 이동속도 | `stats.moveSpeed` | `chariot.GetCurrentMoveSpeed()` |
| 공격 대상 | `ChariotStats targetStats` | `Chariot targetChariot` |
| 투사체 충돌 | targetId/ChariotStats 기반 | `LayerMask` 기반 물리 충돌 |

### 투사체(Projectile) LayerMask 전환
- `Launch()` 시그니처: `(pos, target, dmg, targetId, enemyManager)` / `(pos, target, dmg, ChariotStats)` → `(pos, target, dmg, LayerMask)`
- ArcherView, LancerView 모두 `[SerializeField] LayerMask targetLayers` 추가

### 공격력 계산 변경
- ArcherView/LancerView: 무기 기본위력(`GetBasePower()`) 합산 추가
- 기존: `atk * (1 + skill * 0.01)`
- 변경: `(atk + basePower) * (1 + skill * 0.01)`

### Spec 업그레이드 메서드 추가
| 클래스 | 메서드 |
|--------|--------|
| ArmorSpec | `ApplyDefenseUpgrade()`, `ApplyDurabilityUpgrade()` |
| HorseSpec | `ApplySpeedUpgrade()`, `ApplyPullUpgrade()` |
| WheelSpec | `ApplyAccelUpgrade()`, `ApplyCollisionUpgrade()` |

### GameState 체크 추가
- ChariotCombat, BattleSpawner: `GameStateManager.Instance.CurrentState != GameState.Battle` 시 이동/공격/스폰 중단

### 신규 파일

| 파일 | 내용 |
|------|------|
| GameStateManager.cs | 게임 상태 관리 (Battle 등) |
| ExperienceSystem.cs | 경험치 시스템 |

| Assets/Scripts/Upgrade/ | 업그레이드 시스템 폴더 |

### 적 시스템 개선

- EnemyManager: `OnEnemyDied` 이벤트(경험치 전달), `EnemyHPBarManager` 연동, `ResolveChariot()` 추가

### 핵심 아키텍처 변경: HP/전투 로직을 Chariot 모델로 이관

기존에 ChariotStats(MonoBehaviour)가 HP, 방어력, 사망 처리를 담당했으나, 순수 C# 클래스인 Chariot으로 이관.

| 항목 | 기존 (ChariotStats) | 변경 후 (Chariot) |
|------|---------------------|-------------------|
| HP 관리 | `ChariotStats.currentHP` | `Chariot.GetCurrentHP()` |
| 데미지 처리 | `ChariotStats.TakeDamage()` | `Chariot.TakeDamage()` (방어력 차감 포함) |
| 사망 처리 | `ChariotStats.OnDeath()` 하드코딩 | `Chariot.onDeath` 콜백 |
| 이동속도 | `ChariotStats.moveSpeed` 프로퍼티 | `Chariot.GetCurrentMoveSpeed()` 직접 호출 |
| 무적모드 | 없음 | `Chariot.godMode` 추가 |

### ChariotStats 역할 변경
- 하드웨어 스펙(말/갑옷/바퀴) 보유 + Chariot 조립 전용으로 변경
- ChariotCrewController에 있던 하드웨어 필드를 ChariotStats로 이동
- ChariotCrewController는 승무원 Build만 담당

### 적 전차 ChariotStats 직접 참조 제거
- EnemyChariot, EnemyChariotCombat, EnemyManager → ChariotStats 대신 Chariot 모델 직접 사용
- `ICrewCombat.ExecuteAttack(targetPos, ChariotStats)` → `ExecuteAttack(targetPos, Chariot)`

### 설계 방침: 원거리 데미지는 풀링 투사체 전용
- 궁병/창병의 원거리 공격은 반드시 ObjectPool 투사체를 통해서만 데미지 적용
- 풀이 없거나 투사체 획득 실패 시 폴백 직접 데미지 비활성화
- 근거리(검병 베기, 창병 찌르기)는 즉시 데미지 유지 (추후 이펙트 전환 예정)

### 추후 검토: 섹션별 데미지 시스템
- 전차를 몸체/말/바퀴 섹션으로 분리하여 피격 부위별 차등 데미지
- 각 섹션에 자식 GameObject + Collider2D 배치
- 섹션 식별 컴포넌트(HitboxSection) + 부모 ColliderManager로 피격 분기
- Projectile 도착 시 직접 데미지 → Collider2D 물리 충돌(`OnTriggerEnter2D`) 방식으로 전환 필요
- `Chariot.GetCollisionDamage()` 모델 로직은 존재하나 미연결 상태

### 구조 변경: Before → After
```
Before:
  ChariotCombat이 공격 방식(투사체/근접)까지 전부 관리
  EnemyChariot 하나에 풀링 + 전투 + 자체 타이머 뭉침
  투사체 없음 (즉시 데미지)

After:
  View가 ICrewCombat으로 자기 공격 방식을 캡슐화
  지휘관(ChariotCombat/EnemyChariotCombat)은 "공격해" 명령만
  Projectile로 포물선 화살/투창 발사
  플레이어/적 완전 대칭 구조
  적 투사체는 EnemyManager 공유 풀 (100마리+ 최적화)
```

## 전투 시스템 (Combat/)

| 클래스 | 역할 |
|--------|------|
| **BattleManager** | 중앙 허브. EnemyManager → ChariotCombat 주입 |
| **ChariotCombat** | 전차 지휘관. 배타적 권역 판정 → TryAttack(ICrewCombat) |
| **ICrewCombat** | 병종 공격 인터페이스. 플레이어→적, 적→플레이어 양방향 |
| **Projectile** | 범용 포물선 투사체. ObjectPool 연동 |
| **IChariotCombat** | 아군/적 전차 공통 인터페이스 (GetZoneBoundaries) |
| **ChariotCrewController** | 승무원 View 조립 → ChariotStats에 전달 |
| **ChariotStats** | 하드웨어 스펙 보유 + Chariot 조립. 런타임 전투는 Chariot이 담당 |

**공격 파이프라인**:
```
ChariotCombat.Update()
  → TryGetClosest() (적 전차 탐색)
  → 배타적 권역 판정 (검병/창병/궁병)
  → TryAttack(ICrewCombat) → crew.ExecuteAttack()
    → ArcherView: 화살 Projectile 발사
    → LancerView: 원거리 투창 / 근거리 찌르기
    → SwordsmanView: 즉시 베기
```

## 무게/속도 시스템

```
총 무게 = 말 + 갑옷 + 바퀴 + (궁병 체중+활) + (창병 체중+창) + (검병 체중+검) + (마부 체중)

하중비 = 총 무게 / 말 견인력
  <= 1.0 → 속도 소폭 감소 (최대 -20%)
  > 1.0  → 급격 감소 (최대 -80%)

최종 속도 = 말 기본속도 * 하중패널티 * (1 + 마부숙련 * 0.02)
```
