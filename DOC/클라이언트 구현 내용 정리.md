# 클라이언트 구현 내용 정리

## 폴더 구조

```
Assets/Scripts/
├── Chariot/     ← 전차 하드웨어 데이터 모델
├── Crew/        ← 승무원 성장 데이터 모델
├── Combat/      ← 전투 시스템 (BattleManager, ChariotCombat, ChariotStats)
├── View/        ← 승무원 GameObject 래퍼 (씬 배치 + 쿨타임/데미지/사거리)
├── Enemy/       ← 적 전차 시스템 (EnemyChariot, EnemyManager, BattleSpawner)
├── Core/        ← 유틸리티 (ObjectPool, Camera)
└── StatDebugUI  ← 디버그용
```

---

## 1. 전차 하드웨어 (Chariot/)

| 클래스 | 역할 |
|--------|------|
| **Chariot** | 전차 통합 모델. 무게/속도/회피/충돌데미지 계산 |
| **HorseSpec** | 말 스펙 (견인력, 기본속도, 무게) |
| **ArmorSpec** | 갑옷 스펙 (내구도, 방어력, 무게) |
| **WheelSpec** | 바퀴 스펙 (가속도, 충돌데미지, 무게) |
| **WeaponSpec** | 무기 스펙 (사거리, 기본위력, 무게) |
| **ChariotRole** | 역할 enum (마부/궁병/창병/검병) |

## 2. 승무원 성장 모델 (Crew/)

| 클래스 | 역할 | 고유 스킬 |
|--------|------|-----------|
| **CrewMemberBase** | 추상 베이스. 레벨/경험치/기본스탯 | - |
| **CoachmanPlayer** | 마부. 전차 기동 담당 | ChariotHandlingSkill (+0.2/레벨업) |
| **ArcherPlayer** | 궁병. 원거리 공격 | ArcherySkill (+0.15/레벨업), 활 장착 |
| **LancerPlayer** | 창병. 중거리 공격 | LanceSkill (+0.12/레벨업), 창 장착 |
| **SwordsmanPlayer** | 검병. 근거리 공격 | SwordSkill (+0.12/레벨업), 검 장착 |
| **ChariotCrew** | 4명 컨테이너, 총 무게 계산 | - |

**성장 공식**: `ExpToNextLevel = 50 * Level^1.20`

## 3. 전투 시스템 (Combat/)

| 클래스 | 역할 |
|--------|------|
| **BattleManager** | 중앙 허브. EnemyManager → ChariotCombat 주입 |
| **ChariotCombat** | 전차 이동 + 승무원 공격 관리. 적 1번 탐색 → 3명 사거리 체크 → 공격 |
| **ChariotCrewController** | 하드웨어+승무원 View 조립 → Chariot 모델 빌드 → ChariotStats에 반영 |
| **ChariotStats** | 전차 HP/방어/이동속도. Chariot 모델과 실시간 연동 |

**공격 파이프라인**:
```
BattleManager.Awake() → ChariotCombat.Init(enemyManager)
ChariotCombat.Update()
  → TryGetClosest() 1회 (적 전차 탐색)
  → 궁병: dist <= range && ready → ApplyDamage
  → 창병: dist <= range && ready → ApplyDamage
  → 검병: dist <= range && ready → ApplyDamage
```

## 4. 승무원 View (View/)

| 클래스 | 공격주기 | 사거리 공식 | 데미지 공식 |
|--------|---------|------------|------------|
| **ArcherView** | 1.2s | baseRange * (1 + skill * 0.01) | atk * (1 + skill * 0.01) |
| **LancerView** | 1.0s | baseRange * (1 + skill * 0.01) | atk * (1 + skill * 0.01) |
| **SwordsmanView** | 0.6s | range * (1 + skill * 0.005) | (atk + basePower) * (1 + skill * 0.04) |
| **CoachmanView** | - | - | 전투 없음, 이동속도에 기여 |

각 View는 `IsReady()` / `ConsumeAttack()` / `GetEffectiveRange()` / `GetDamage()` 인터페이스 제공

## 7. 무게/속도 시스템

```
총 무게 = 말 + 갑옷 + 바퀴 + (궁병 체중+활) + (창병 체중+창) + (검병 체중+검) + (마부 체중)

하중비 = 총 무게 / 말 견인력
  <= 1.0 → 속도 소폭 감소 (최대 -20%)
  > 1.0  → 급격 감소 (최대 -80%)

최종 속도 = 말 기본속도 * 하중패널티 * (1 + 마부숙련 * 0.02)
```

- **플레이어**: 각 View가 `[SerializeField]`로 자기 풀 직접 소유 (인스턴스 1개뿐)
